# Security Policy

## Supported Versions

Routa is currently pre-1.0. Security fixes are applied to the latest published version unless maintainers explicitly announce support for older versions.

## Reporting a Vulnerability

Do not open a public issue for security vulnerabilities.

Report suspected vulnerabilities privately to `security@routa.dev`.

Please include:

- affected package and version
- vulnerability description
- reproduction steps or proof of concept
- expected impact
- any known mitigations

## Response

Maintainers will acknowledge reports as soon as practical, investigate the issue, and coordinate a fix and disclosure timeline when the report is valid.

## Scope

Security reports should relate to Routa packages, generated project behavior, CLI behavior, or documentation that could cause users to deploy insecure systems.
