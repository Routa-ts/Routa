`@routa-ts/core` provides the route definition helpers and runtime primitives used by Routa applications and generated code.

## Install

```sh
pnpm add @routa-ts/core hono zod
```

## Basic Usage

```ts
import { createRoute, createRouteRoot } from "@routa-ts/core";
import { z } from "zod";

const route = createRouteRoot("/status");

export default route({
	get: createRoute({
		responses: {
			success: {
				status: 200,
				schema: z.object({
					ok: z.boolean(),
				}),
			},
		},
		run: async () => {
			return { type: "success", data: { ok: true } };
		},
	}),
});
```

## Model

Route contracts keep HTTP behavior explicit in source: inputs, responses, middleware, metadata, and handler boundaries live together.

Business logic remains application-owned. Routa owns the HTTP boundary.

## Main APIs

- `createRouta(config)`: preserves the app runtime configuration.
- `createRoute(contract)`: preserves a method contract with typed input, responses, middleware, and handler output.
- `createMiddleware(contract)`: defines middleware requirements, provided context, rejects, and runtime behavior.
- `createRouteRoot(path)`: returns a path-bound `route(config)` helper that binds each method to its generated context type.
- `createHonoApp(routes, options)`: creates the Hono runtime app from generated route metadata.
- `createLogger(options)`: creates Routa's built-in backend-neutral structured logger.

## Logger Contract

`RoutaLogger` is a structural Routa type with no logging-package dependency. Its level methods are `trace`, `debug`, `info`, `warn`, `error`, `fatal`, and the `silent` no-op. It also provides `log(event)`, `child(bindings)`, `bindings()`, and `isLevelEnabled(level)`.

Configure any implementation with `createRouta({ logger })`; route handlers receive that same implementation as `ctx.logger`. `logger: false` supplies the complete no-op contract instead.

The full API example includes an opt-in route and local exercise script covering every method without making alert-level demonstration logs part of ordinary endpoint traffic.

<Aside type="note" title="Generated types">
	Generated `.routa/routes.gen.ts` files augment core types so route context can be inferred from middleware metadata.
</Aside>
