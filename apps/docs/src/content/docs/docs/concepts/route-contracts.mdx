---
title: Route Contracts
description: Routa route files define the HTTP contract in source.
---

Routa is built around one source of truth:

```txt
schemas -> routes -> handlers -> OpenAPI -> docs/checks
```

Route files may use either directory style (`routes/users/route.ts`) or flat style (`routes/users.ts`). Dynamic flat files use `$`, for example `routes/users.$id.ts` maps to `/users/:id`; both styles can be mixed in one project.

Route files define what each HTTP method accepts, what it can return, which middleware applies, and where the handler boundary begins.

## File-Based APIs

Folders map to URL segments:

```txt
src/routes/users/route.ts
src/routes/users/$userId/route.ts
src/routes/users/$userId/posts/route.ts
```

Each `route.ts` file owns the methods for one path. Keeping all verbs for one path together makes shared params, middleware, schemas, and metadata easier to maintain.

Applications may declare `get`, `post`, `put`, `patch`, `delete`, and `head`. Do not declare `options`: Routa generates a `204 OPTIONS` response from the path's methods and keeps its `Allow` header in sync automatically.

## Contract Before Handler

```ts
const route = createRouteRoot("/users");

export default route({
	post: createRoute({
		input: {
			body: CreateUserSchema,
		},
		responses: {
			success: {
				status: 201,
				schema: UserSchema,
			},
			emailConflict: {
				status: 409,
				schema: EmailConflictSchema,
			},
		},
		run: async ({ input, ctx }) => {
			ctx.logger.info("users.create", "Creating a user.", {
				actorId: ctx.auth.userId,
			});

			return {
				type: "success",
				data: await users.createUser({
					body: input.body,
					actorId: ctx.auth.userId,
				}),
			};
		},
	}),
});
```

Handlers return named outcomes from the declared response map. If a route did not declare an outcome, TypeScript should reject it.

Every handler also receives the configured Routa logger as `ctx.logger`. The value implements Routa's backend-agnostic `RoutaLogger` contract and is the same logger used for runtime lifecycle events. It exposes `trace`, `debug`, `info`, `warn`, `error`, `fatal`, and a `silent` no-op, together with child bindings and level checks. If application configuration sets `logger: false`, Routa supplies a typed no-op logger with that complete surface so handler code remains configuration-independent.

## HTTP Boundary Only

Routa owns routing, validation, middleware context, typed responses, and OpenAPI. Your application owns services, use cases, domain models, persistence, and business logic.
