Routa writes generated files into `.routa/`.

## `.routa/routes.gen.ts`

Generated route metadata used by the runtime, CLI checks, and type augmentation.

## `.routa/manifest.json`

Tracks generated files, source operation IDs, file kinds, and hashes. The manifest lets Routa detect whether regeneration would update, preserve, conflict with, or remove files.

## `.routa/openapi-baseline.json`

Stores the OpenAPI contract used for drift checks and removed-operation checks.

Commit these files. They make project behavior reproducible across machines and CI.

## Workflow

Use `<package-manager> run generate` after changing route files, route methods, or
middleware layout. Review and commit the resulting `.routa/` diff with the source change.
`<package-manager> run check`, `<package-manager> run build`, and
`<package-manager> run dev` also synchronize metadata after successful validation, so they
can legitimately change the working tree.
