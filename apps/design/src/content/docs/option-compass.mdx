## Callouts as bearings

<Aside type="note" title="Note · Steady bearing">
	Route folders map to URL segments. Keep the tree readable and the path stays true.
</Aside>

<Aside type="tip" title="Tip · Fix your heading">
	Start from OpenAPI when the map already exists — scaffold, then refine contracts.
</Aside>

<Aside type="caution" title="Caution · Magnetic drift">
	Hand-edited generated files will diverge. Regenerate from the source contract instead.
</Aside>

<Aside type="danger" title="Danger · Off the chart">
	Shipping without `routa check` leaves OpenAPI and handlers out of sync.
</Aside>

## Course plot · Steps

<Steps>

1. Create a project.

	```sh
	pnpm create routa-ts@latest
	```

2. Define a route contract in `src/routes`.

	```ts
	export default createRouteRoot("/status")({
		get: createRoute({
			responses: {
				success: { status: 200, schema: StatusSchema },
			},
			run: async () => ({ ok: true }),
		}),
	});
	```

3. Check alignment, then start the server.

	```sh
	pnpm routa check
	pnpm routa start
	```

</Steps>

## Contract sample

```ts
export default createRouteRoot("/users")({
	post: createRoute({
		input: { body: CreateUserSchema },
		responses: {
			success: { status: 201, schema: UserSchema },
			conflict: { status: 409, schema: ErrorSchema },
		},
		run: async ({ input, ctx }) => {
			return users.createUser(input.body, ctx);
		},
	}),
});
```

> Compass identity: **orient first, then move.** Docs should feel like instruments — clear bearings, no fog.
