## Structural asides

<Aside type="note" title="Note · Cell bound">
	Every success path is a named key in `responses` — not a free-form status code.
</Aside>

<Aside type="tip" title="Tip · Share schemas">
	Colocate `schemas.ts` beside `route.ts` so the lattice stays local and reviewable.
</Aside>

<Aside type="caution" title="Caution · Loose node">
	Optional fields without defaults can widen the OpenAPI surface unexpectedly.
</Aside>

<Aside type="danger" title="Danger · Broken edge">
	Returning an undeclared outcome bypasses the type lattice and fails the check.
</Aside>

## Assembly steps

<Steps>

1. Initialize the chassis.

	```sh
	pnpm create routa-ts@latest
	```

2. Define a typed cell.

	```ts
	const CreateUserSchema = z.object({
		email: z.string().email(),
		name: z.string().min(1),
	});

	export default defineRoute({
		post: createRoute({
			input: { body: CreateUserSchema },
			responses: {
				success: { status: 201, schema: UserSchema },
			},
			run: async ({ input, ctx }) => users.createUser(input.body, ctx),
		}),
	});
	```

3. Verify edges, then serve.

	```sh
	pnpm routa check
	pnpm routa start
	```

</Steps>

## Response map

```ts
responses: {
	success: { status: 200, schema: UserSchema },
	notFound: { status: 404, schema: ErrorSchema },
	conflict: { status: 409, schema: ErrorSchema },
},
```

| Outcome key | Status | Schema |
| --- | --- | --- |
| `success` | 200 | `UserSchema` |
| `notFound` | 404 | `ErrorSchema` |
