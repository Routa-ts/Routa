## Marginalia · Asides

<Aside type="note" title="Note · Entry recorded">
	A `route.ts` file owns the methods for its path. Treat it as the ledger line for that URL.
</Aside>

<Aside type="tip" title="Tip · Prefer the source deed">
	When OpenAPI and source disagree, regenerate from the contract you intend to keep.
</Aside>

<Aside type="caution" title="Caution · Unsealed change">
	Editing `.routa/` by hand breaks the chain of custody between source and generated types.
</Aside>

<Aside type="danger" title="Danger · Broken seal">
	Deploying without a passing `routa check` ships an unsigned boundary.
</Aside>

## Filing steps

<Steps>

1. Open a new ledger (project).

	```sh
	pnpm create routa-ts@latest
	```

2. Enter the first line item — a route contract.

	```ts
	export default defineRoute({
		get: createRoute({
			responses: {
				success: { status: 200, schema: UserSchema },
			},
			run: async ({ ctx }) => users.list(ctx),
		}),
	});
	```

3. Countersign with check, then serve.

	```sh
	pnpm routa check && pnpm routa start
	```

</Steps>

## Sample deed

```ts
export default defineRoute({
	put: createRoute({
		input: {
			params: TenantParams,
			body: UpdateTenantSchema,
		},
		responses: {
			success: { status: 200, schema: TenantSchema },
			notFound: { status: 404, schema: ErrorSchema },
		},
		run: async ({ input, ctx }) => {
			return tenants.update(input.params.tenantId, input.body, ctx);
		},
	}),
});
```

> Ledger identity: **write it once, seal it, keep it true.** Docs should feel like a well-kept archive — copper, stone, and ink.
